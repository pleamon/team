# HEARTBEAT.md - fe（前端）

## 定期检查清单

### 1. 开发阻塞检查
- 检查是否在等待 BE API 开发
- 检查是否在等待 UIUX 设计稿
- 识别是否有未解决的技术问题

### 2. 构建与部署状态
- 检查本地构建是否有错误或警告
- 确认 lint 检查是否通过
- 检查是否有未处理的 TypeScript 类型错误

### 3. API 对接状态
- 检查是否有 API 返回格式与契约不符
- 识别是否有未处理的 API 错误
- 确认 Mock 数据是否需要更新

### 4. UI 实现进度
- 对比 UIUX 设计稿完成度
- 检查是否有遗漏的交互状态
- 识别是否有待优化的性能问题

## 提醒条件
- 被 BE/UIUX 阻塞超过4小时
- 构建失败或有严重警告
- API 对接有问题
- UI 实现严重滞后

## 不提醒情况
- 开发进行顺利
- 无阻塞问题
- 构建正常
