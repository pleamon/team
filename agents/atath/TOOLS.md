# TOOLS.md - Atath

## 核心工具

### 1. Slack 通信
所有任务下达和状态同步通过 Slack。

- 使用 `<@UXXXXXXXX>` 触发 Agent 响应
- 使用文本名称仅提及不触发

### 2. Git 文档协作
PRD、Sprint 计划等文档通过 Git 管理。

```bash
# 提交文档
git add -A && git commit -m "doc: <描述>"
git push
```

---

## 文档模板

### PRD 模板
```markdown
# PRD: [功能名称]

## 概述
[一句话描述这个功能]

## 背景
[为什么需要这个功能]

## 用户故事
作为 [用户角色]，我想要 [功能]，以便 [价值]

## 功能需求
### 必须有 (P0)
- [ ] 需求1
- [ ] 需求2

### 应该有 (P1)
- [ ] 需求3

## 验收标准
- [ ] 标准1
- [ ] 标准2

## 非功能需求
- 性能：[要求]
- 兼容性：[要求]
```

### Sprint 计划模板
```markdown
# Sprint [N] 计划

**周期**：YYYY-MM-DD ~ YYYY-MM-DD
**目标**：[Sprint 目标]

## 任务列表

| ID | 任务 | 负责人 | 依赖 | 状态 |
|----|------|--------|------|------|
| T1 | ... | FE | - | 🚧 |
| T2 | ... | BE | T1 | ⏳ |

## 风险
- [风险1]：[应对措施]
```

---

## 状态标记

| 标记 | 含义 |
|------|------|
| ✅ | 已完成 |
| 🚧 | 进行中 |
| ⏳ | 等待依赖 |
| 🔴 | 阻塞 |

## 优先级

| 级别 | 含义 |
|-----|------|
| P0 | 必须有 |
| P1 | 应该有 |
| P2 | 可以有 |
| P3 | 暂不做 |
