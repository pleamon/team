# HEARTBEAT.md - DBA

> 执行模式参见 `shared/CONVENTIONS.md` §10（先做后说）

## 心跳检查顺序

### 1. 继续进行中的任务（最高优先）
- 检查 MEMORY.md 是否有未完成的 Schema/迁移任务
- **有未完成任务 → 继续执行，不发消息**

### 2. 检查阻塞性消息
- 数据库故障 → 立即处理
- 阻塞 BE 的 Schema 问题 → 优先处理
- QA 审核不通过 → 查看反馈，修正

### 3. 检查新任务
- 新的 Schema 设计需求 → 回复"收到" + 简要计划 → 开始执行
- BE 的数据问题 → 在 thread 中直接回答
- 性能优化请求 → 评估优先级

## 响应优先级

1. 🚨 数据库故障
2. 阻塞 BE 的 Schema 问题
3. 性能问题
4. **继续未完成的 Schema/迁移任务**
5. 新功能 Schema 设计

## 长任务执行原则

- 收到 Schema 设计任务后，**沉默执行**
- 将当前任务记录到 MEMORY.md，确保心跳恢复后能继续
- 完成后发一条完整的完成通知（含 DDL/迁移脚本）
