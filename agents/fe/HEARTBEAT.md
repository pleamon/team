# HEARTBEAT.md - FE

> 执行模式参见 `shared/CONVENTIONS.md` §10（先做后说）

## 心跳检查顺序

### 1. 继续进行中的任务（最高优先）
- 检查 MEMORY.md 是否有未完成的开发任务
- **有未完成任务 → 继续执行，不发消息**
- 只在有实质进展时更新任务 thread：`🚧 进度：[已完成] / [计划]`

### 2. 检查阻塞性消息
- 线上 Bug → 立即处理
- QA 审核不通过 → 查看反馈，开始修复
- 接口变更影响当前工作 → 评估影响

### 3. 检查新任务
- 新的 @mention 开发任务 → 回复"收到" + 简要计划 → 开始执行
- UIUX 的设计更新 → 评估是否影响当前工作
- BE 的接口变更通知 → 评估是否影响当前工作

## 响应优先级

1. 🚨 线上 Bug
2. QA 反馈的 Bug
3. 阻塞其他人的任务
4. **继续未完成的开发任务**
5. 新功能开发

## 长任务执行原则

- 收到开发任务后，**沉默执行**，专注写代码
- 将当前任务记录到 MEMORY.md，确保心跳恢复后能继续
- 每次心跳时检查进度，有实质进展才在 thread 更新
- 完成后发一条完整的完成通知（含 commit/产出路径）
- **不要每做完一小步就发消息**
