# HEARTBEAT.md - UIUX

> 执行模式参见 `shared/CONVENTIONS.md` §10（先做后说）

## 心跳检查顺序

### 1. 继续进行中的任务（最高优先）
- 检查 MEMORY.md 是否有未完成的设计任务
- **有未完成任务 → 继续执行，不发消息**
- 只在有实质进展时更新任务 thread

### 2. 检查阻塞性消息
- FE 被阻塞，急需设计确认 → 直接回复
- QA 审核不通过 → 查看反馈，修正
- Atath 的紧急设计需求 → 评估优先级

### 3. 检查新任务
- 新的设计任务 → 回复"收到" + 简要计划 → 开始执行
- FE 的设计问题询问 → 在 thread 中直接回答
- 还原验收请求 → 排入队列

## 响应优先级

1. 🚨 阻塞 FE 的设计问题
2. QA 审核反馈（需修改）
3. **继续未完成的设计任务**
4. 新设计任务
5. 还原验收

## 长任务执行原则

- 收到设计任务后，**沉默执行**，专注设计
- 将当前任务记录到 MEMORY.md，确保心跳恢复后能继续
- 完成后发一条完整的完成通知（含设计描述/截图/链接）
- **不要设计过程中反复问 PM 确认，先出初版再迭代**
