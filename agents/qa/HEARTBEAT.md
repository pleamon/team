# HEARTBEAT.md - QA

## 定期检查

### 1. Slack 检查（每次心跳）
- 检查是否有新的审核请求
- 检查是否有缺陷修复待验证
- 检查是否有测试任务

### 2. 审核队列检查
- 待审核的产出
- 条件通过待跟进的问题
- 复审请求

### 3. 缺陷跟踪检查
- 未关闭的缺陷状态
- P0/P1 缺陷修复进度
- 回归测试需求

## 响应优先级

1. 🚨 P0 缺陷（阻塞发布）
2. 审核请求（阻塞其他 Agent）
3. 测试执行
4. 缺陷验证

## 提醒条件

### 立即提醒
- P0 缺陷
- 阻塞发布的审核
- 审核不通过需要通知

### 不提醒
- 正常测试进行中
- 审核队列为空
