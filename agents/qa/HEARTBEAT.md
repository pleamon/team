# HEARTBEAT.md - QA

> 执行模式参见 `shared/CONVENTIONS.md` §10（先做后说）

## 心跳检查顺序

### 1. 继续进行中的任务（最高优先）
- 检查 MEMORY.md 是否有未完成的审核/测试任务
- **有未完成任务 → 继续执行，不发消息**

### 2. 检查阻塞性消息
- P0 缺陷 → 立即跟进
- 阻塞发布的审核 → 优先处理

### 3. 检查审核队列
- 新的审核请求 → 回复"收到，开始审核" → 开始执行
- 缺陷修复待验证 → 执行回归测试
- 复审请求 → 排入队列

### 4. 缺陷跟踪检查
- 未关闭的缺陷状态
- P0/P1 缺陷修复进度

## 响应优先级

1. 🚨 P0 缺陷（阻塞发布）
2. 审核请求（阻塞其他 Agent）
3. **继续未完成的审核/测试任务**
4. 测试执行
5. 缺陷验证

## 长任务执行原则

- 收到审核任务后，**沉默执行**，逐项检查
- 将当前任务记录到 MEMORY.md，确保心跳恢复后能继续
- 完成后发一条完整的审核报告
- **不要审到一半就发消息问问题，先完成整体审核再汇总反馈**
